# Lot 5.2 — Tableau de bord budget vs réalisé

> **Statut** : Livré le 2026-05-07
> **Branche backend** : `lot-5/5.2-tableau-bord`
> **Branche frontend** : `lot-5/5.2-tableau-bord`

## 1. Objectif métier

Restituer aux gestionnaires de budget une vue agrégée des **écarts**
entre le budget validé (`fait_budget` sur version au statut `gele`) et
le réalisé consolidé (`fait_realise` au statut `VALIDE`), avec :

- 4 KPI synthétiques (total écarts, critique, attention, écart absolu)
- Un tableau détaillé ligne à ligne (CR × compte × ligne métier × mois)
- Un export Excel multi-onglets pour les retours métier hors plateforme

Le tout filtrable et borné au **périmètre** de l'utilisateur connecté
(`user_perimetres` Lot 4.1, fallback `bridge_user_role`).

## 2. Règles de calcul

### 2.1 Niveaux d'alerte (paramétrables)

L'utilisateur fixe deux seuils en pourcentage : `seuilEcartPctAttention`
(défaut 5 %) et `seuilEcartPctCritique` (défaut 10 %).

| Situation | Niveau |
|---|---|
| `montantRealise IS NULL` | `MANQUANT` |
| `\|ecartPct\| ≥ seuilCritique` | `CRITIQUE` |
| `\|ecartPct\| ≥ seuilAttention` | `ATTENTION` |
| sinon | `NORMAL` |

`ecartPct` = `(realise − budget) / budget * 100`. Si `budget = 0` et
`realise > 0`, ecartPct est `null` mais le niveau passe à `CRITIQUE`
(dépassement infini).

### 2.2 Sens de l'écart (UEMOA)

Dépend de la **classe comptable** du compte (`dim_compte.classe`) :

| Classe | Nature | Sens favorable |
|---|---|---|
| `6` | CHARGE | `realise < budget` (sous-consommation) |
| `7` | PRODUIT | `realise > budget` (sur-collecte) |
| autre | BILAN | toujours `NEUTRE` |

Implémentation : `analyse-ecarts.service.ts` helpers `classeToNature` et
`sensEcartFor`.

### 2.3 Stratégie SQL

Une seule requête avec `LEFT JOIN` (`fait_budget` à gauche) pour
conserver les lignes **MANQUANT** (budget existe, réalisé absent) sans
faire deux passes.

```sql
FROM fait_budget fb
INNER JOIN dim_compte c ON c.id = fb.fk_compte
INNER JOIN dim_centre_responsabilite cr ON cr.id = fb.fk_centre
LEFT JOIN fait_realise fr
  ON fr.fk_centre_responsabilite = fb.fk_centre
 AND fr.fk_compte = fb.fk_compte
 AND fr.fk_ligne_metier = fb.fk_ligne_metier
 AND fr.fk_temps = fb.fk_temps
 AND fr.statut = 'VALIDE'
WHERE fb.fk_version = $1
  AND fb.fk_scenario = $2
  ...
ORDER BY ABS(COALESCE(fr.montant - fb.montant, 0)) DESC
```

> Note convention : `fait_budget.fk_centre` ↔ `fait_realise.fk_centre_responsabilite`.
> Le SELECT renomme via `fb.fk_centre AS fk_centre_responsabilite` pour
> uniformiser le DTO côté API.

## 3. API backend

### 3.1 `GET /tableau-de-bord/budget-vs-realise`

Permissions : `BUDGET.LIRE` **et** `REALISE.LIRE` (double via
`@RequirePermissions({ all: [...] })`).

Query :

| param | type | requis | défaut |
|---|---|---|---|
| `versionId` | uuid | oui | — |
| `scenarioId` | uuid | oui | — |
| `crIds` | uuid[] | non | toutes du périmètre |
| `ligneMetierIds` | uuid[] | non | toutes |
| `moisDebut` | `YYYY-MM` | oui | — |
| `moisFin` | `YYYY-MM` | oui | — |
| `seuilEcartPctAttention` | number | non | 5 |
| `seuilEcartPctCritique` | number | non | 10 |

Réponse : `EcartsResponseDto` (`filtres`, `kpi`, `lignes[]`).

### 3.2 `GET /tableau-de-bord/budget-vs-realise/export`

Mêmes filtres, renvoie un fichier `.xlsx` (3 onglets : Synthèse / Détail
des écarts / Filtres). Couleurs conditionnelles sur la colonne `Niveau`
selon le tableau §2.1.

## 4. Frontend

### 4.1 Route et navigation

- Route `/tableau-de-bord/budget-vs-realise` (lazy-loaded)
- Sidebar : groupe « Exécution », entrée « Tableau de bord » (icône `BarChart3`)
- Garde `PermissionRoute` sur `REALISE.LIRE` (le composant masquera l'export si manque `BUDGET.LIRE`).

### 4.2 Sections de la page

1. **Filtres** (`FiltresEcartsForm`) : version (auto-sélection version `gele`), scénario, période, CRs, seuils. Validation client : `moisFin ≥ moisDebut`, `seuilCritique > seuilAttention`.
2. **KPI cards** (`KpiCardsRow`) : 4 cards horizontales.
3. **Tableau** (`EcartsTable`) : tri cliquable par colonne, mise en forme conditionnelle par niveau, badges nature/niveau, icône sens.

Filtres rapides en tête du tableau : Tous / Critiques / Attention /
Manquants. Recherche texte sur `codeCr` + `codeCompte`. Empty state
explicite si aucune ligne.

### 4.3 Persistance

`tableau-bord-store` (Zustand + persist) sauvegarde les filtres
principaux dans `localStorage` (`tableau-bord-store-v1`) pour les
retrouver à l'ouverture suivante. L'état volatile (`ecarts`, `loading`,
`error`, filtre rapide, recherche) n'est pas persisté.
